-   [A List Apart: Articles: Building Twitter
    Bootstrap](http://www.alistapart.com/articles/building-twitter-bootstrap/)\
    Det här projektet är verkligen något att drömma om. Bootstrap är det
    förata CSS-biblioteket som fått godkänt under min lupp. Det är några
    riktigt skillade personer som gjort sitt yttersta i detta. Även en
    inspirerande artikel som är värd att läsa för att se hur hållbar och
    bulletproof CSS skrivs i verkligheten.
-   [Coding Horror: Meetings: Where Work Goes to
    Die](http://www.codinghorror.com/blog/2012/02/meetings-where-work-goes-to-die.html)\
    Ännu en artikel på ämnet \"möten dödar produktivitet\".
-   [HTML5 Rocks - Getting Started With the Track
    Element](http://www.html5rocks.com/en/tutorials/track/basics/)\
    Visa subtitles för videor enligt HTML5-standarden. Jag ser framför
    mig ljusa år för tillgänglighet och användbarhet i detta.
-   [twolfson/File-Watcher -
    GitHub](https://github.com/twolfson/File-Watcher)\
    Verktyg som \"övervakar\" filer i ett webbprojekt och laddar om
    webbläsarfliken när någon av de övervakade filerna har ändrats. Kan
    vara anvndbart för t ex författande av enhetstester för JavaScript
    eller semi-live preview vid intensivt CSS-kodande.
-   [Bricss - Forget loading spinners, use a wait
    cursor](http://bricss.net/post/17941857127/forget-loading-spinners-use-a-wait-cursor)\
    Inte så dumt. Det är mer native och mer universiellt. Önskar att det
    fanns något riktigt \"loading\"-state också, så att webbläsarnas
    egna spinners och loaders gick att tala med.

-   [Best Practice for Programming with Vendor Prefixes »
    SitePoint¬es=](http://www.sitepoint.com/best-practice-for-programming-with-vendor-prefixes/#fbid=oZuimIPS9-s)\
    Mitt tips är att helt överge tanken på att hantera detta med
    JavaScript. Hantera CSS med klasser och applicera genom átt lägga
    till och ta bort klasserna med JavaScript.
-   [Things It Might Be Fun/Useful to Try the Universal (\*) Selector On
    \|
    CSS-Tricks¬es=](http://css-tricks.com/things-it-might-be-funuseful-to-try-the-universal-selector-on/)\
    Sedan jag började med OOCSS har jag några utav dessa exempel i egna
    klasser, t ex .smooth for att enkelt kunna applicera transitions och
    spara några viktiga bytes i CSS-filen. Universal selector känns
    litet riskabelt och jag föredrar átt hålla dessa CSS-deklarationer
    sparsamma.
